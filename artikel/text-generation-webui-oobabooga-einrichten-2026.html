Text Generation WebUI (oobabooga) einrichten – Setup-Guide 2026

Lokale LLMs auf dem eigenen Server oder Desktop: Installation, Modell-Loading, LoRA-Integration, API-Mode und Extensions. Der vollständige Guide zum leistungsstärksten LLM-Interface 2026.

— Anzeige —

Google AdSense Platzhalter

Text Generation WebUI – auch oobabooga genannt – ist das Schweizer Taschenmesser für lokale Large Language Models (LLMs). Die Plattform vereint grafische Oberfläche, API-Server und ein umfangreiches Extensions-System in einem schlanken Python-Tool. Egal ob Llama 3, Mistral, Gemma, Qwen oder DeepSeek – oobabooga liefert die Infrastruktur, um LLMs lokal und datenschutzkonform zu betreiben.

Anders als Cloud-APIs verlassen bei oobabooga keine Daten deinen Rechner oder VPS. Du behältst die volle Kontrolle über Modell-Auswahl, Quantisierung, Kontextlänge und System-Prompts. Hugging Face stellt Tausende quantisierter Modelle bereit – von winzigen 1B-Parametermodellen für schwache Hardware bis zu leistungsstarken 70B-Modellen für dedizierte GPU-Server.

Dieser Guide führt dich Schritt für Schritt durch die Installation auf Linux und Windows, erklärt die Unterschiede zwischen GGUF, GPTQ und EXL2, zeigt dir den Umgang mit LoRA-Adaptern und die Konfiguration des API-Modes für die Integration in eigene Anwendungen. Dazu kommen die wichtigsten Extensions: TTS für Sprachausgabe, RAG für wissensbasierte Abfragen und Whisper für Spracheingabe. Praxisnah und auf dem Stand von 2026.

🔑 Kurzfazit: oobabooga ist 2026 die ausgereifteste Lösung für lokale LLMs. Mit Unterstützung für GGUF, GPTQ, EXL2, AWQ und HQQ deckt sie alle Quantisierungsformate ab. Der API-Mode bietet eine OpenAI-kompatible REST-Schnittstelle, und Extensions wie TTS, RAG und Whisper machen sie zur vollwertigen KI-Plattform. Für Einsteiger und Fortgeschrittene gleichermaßen geeignet.

Systemanforderungen

  • 7B-Modelle: 8 GB VRAM (GPU) oder 16 GB RAM (CPU/Offloading)
  • 13B–70B-Modelle: 16–48 GB VRAM bzw. 32–64 GB RAM
  • GPU: NVIDIA CUDA wird empfohlen; AMD ROCm und Apple Metal werden unterstützt
  • Betriebssystem: Linux (Ubuntu 22.04/24.04 LTS) oder Windows 10/11
  • Speicher: 20–100 GB freier SSD-Platz pro Modell

Installation – Linux (Ubuntu/Debian)

Linux ist die bevorzugte Plattform für den Server-Betrieb. Die Einrichtung dauert wenige Minuten:

  1. Conda installieren: wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh && bash Miniconda3-latest-Linux-x86_64.sh
  2. Repository klonen: git clone https://github.com/oobabooga/text-generation-webui.git && cd text-generation-webui
  3. Venv einrichten: conda create -n textgen python=3.12 && conda activate textgen
  4. CUDA-Pytorch: pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
  5. Requirements: pip install -r requirements.txt
  6. Starten: python server.py --listen – UI unter http://deine-ip:7860

Installation – Windows

Auf Windows erledigt der start_windows.bat-Installer alles automatisch: Conda, CUDA-Erkennung und Paket-Installation. Einfach das Repository klonen, die Batch-Datei ausführen und warten – nach 5–15 Minuten öffnet sich die Weboberfläche automatisch im Browser.

💡 Tipp: Für den Server-Betrieb auf einem VPS startest du oobabooga mit python server.py --listen --api. Der API-Mode aktiviert die OpenAI-kompatible Schnittstelle auf Port 5000. Mit Nginx-Reverse-Proxy und Let's Encrypt SSL (siehe SSL-Guide) machst du die API sicher über HTTPS erreichbar.

Modelle laden: GGUF, GPTQ und EXL2

oobabooga unterstützt fünf Quantisierungsformate via spezialisierter Loader. Die drei wichtigsten im Überblick:

Format Backend Hardware Geschwindigkeit Ideales Szenario
GGUF llama.cpp CPU + GPU (Hybrid) 20–40 T/s CPU-only, GPU-Offloading, feinste Quantisierungsstufen
GPTQ AutoGPTQ GPU (NVIDIA CUDA) 60–100 T/s Altere GPUs, 4-Bit-Quantisierung, stabile Inferenz
EXL2 ExLlamaV2 GPU (NVIDIA CUDA) 80–160 T/s High-End-GPUs, variable Bitrate, Flash-Attention v2

GGUF von llama.cpp eignet sich für CPU-Betrieb oder Hybrid-Setups mit teilweisem GPU-Offloading. Es bietet die feinste Abstufung der Quantisierungslevel (Q2_K bis Q8_0) und läuft auch ohne dedizierte Grafikkarte. Im Model-Tab wählst du den Loader llama.cpp, gibst den GGUF-Dateipfad an und stellst n-gpu-layers für das Offloading ein. Ab 40–60 GPU-Layern läuft ein 7B-Modell fast vollständig auf der GPU.

GPTQ war lange das Standardformat für GPU-Inferenz. Der Loader AutoGPTQ ermöglicht 4-Bit-Quantisierung mit minimalem Qualitätsverlust und ist besonders effizient auf RTX-3000-Serien-GPUs.

EXL2 (ExLlamaV2) ist 2026 die schnellste Option für reine GPU-Inferenz. Es unterstützt variable Bitraten – verschiedene Layer mit unterschiedlichen Genauigkeiten – und bietet Flash-Attention v2 sowie 8-Bit-Cache. Für 70B-Modelle auf einer 48-GB-GPU ist EXL2 die klare Nummer eins.

Modelle herunterladen

Im Model-Tab der UI findest du den Hugging-Face-Downloader. Alternativ per Kommandozeile:

# GGUF
wget https://huggingface.co/bartowski/Meta-Llama-3.1-8B-Instruct-GGUF/resolve/main/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf -O models/llama-3.1-8b-q4.gguf

# EXL2 per git LFS
git lfs clone https://huggingface.co/turboderp/Llama-3.2-3B-ExL2 models/llama-3.2-3b-exl2

📊 Empfehlung 2026: Für einen GPU-Server mit NVIDIA RTX 4090 (24 GB VRAM) setze auf EXL2 – es liefert die höchste Token-Rate und beste Speicherausnutzung. Für CPU-Betrieb auf einem VPS ohne GPU ist GGUF die einzig sinnvolle Wahl. GPTQ bleibt eine solide Allround-Option für 4-Bit-Inferenz auf Mittelklasse-GPUs wie der RTX 3060 oder A2000.

LoRA-Adapter laden und trainieren

LoRA (Low-Rank Adaptation) passt LLMs an spezifische Aufgaben an – ohne das gesamte Modell zu verändern. Im LoRA-Tab kannst du:

  • Adapter laden: Vortrainierte LoRAs aus dem Hugging Face Hub direkt auf dein Basismodell anwenden.
  • Multi-LoRA: Mehrere Adapter gleichzeitig laden und deren Gewichtung mischen (z. B. 0,7 Stil-LoRA + 0,3 Fach-LoRA).
  • QLoRA-Training: Integrierter Trainer für JSONL-Datensätze im ChatML-Format. Mit 8-Bit-Basismodell reichen 12 GB VRAM für das Training eines 7B-Modells. Trainingszeit für 1.000 Samples bei 3 Epochen auf einer RTX 4090: ca. 30–45 Minuten.
  • Merge: Adapter dauerhaft in das Basismodell mergen.

API-Mode: OpenAI-kompatible REST-Schnittstelle

Der API-Mode ist eine der wertvollsten Funktionen für den VPS-Betrieb. Aktivierung mit --api:

python server.py --listen --api --api-port 5000 \
  --model llama-3.1-8b-q4.gguf \
  --loader llama.cpp \
  --n-gpu-layers 60 \
  --max_seq_len 8192

Die API ist vollständig kompatibel mit OpenAI Chat Completions (v1/chat/completions) und Text Completions (v1/completions). Jedes Tool, das OpenAI unterstützt (LangChain, LlamaIndex, Open WebUI, SillyTavern, Continue.dev, Cursor), kann nahtlos auf dein lokales Modell umgeleitet werden – inklusive Streaming (SSE) und Token-Usage-Reporting. Beispiel mit curl:

curl http://localhost:5000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Meta-Llama-3.1-8B-Instruct",
    "messages": [{"role": "user", "content": "Erkläre oobabooga in drei Sätzen."}],
    "temperature": 0.7,
    "max_tokens": 500
  }'

Mit Nginx-Reverse-Proxy und API-Key-Authentifizierung betreibst du einen vollwertigen KI-API-Dienst für dein Team – datenschutzkonform, weil keine Daten dein Netzwerk verlassen. Besonders wertvoll für Unternehmen mit strengen Compliance-Vorgaben.

Extensions: TTS, RAG und Whisper

Das Extensions-System macht oobabooga zur Komplettlösung:

Text-to-Speech (TTS)

Verwandelt LLM-Ausgabe in gesprochene Sprache. Unterstützt XTTS-v2 (Voice-Cloning mit 6 Sekunden Audiomaterial), ElevenLabs API (100+ Stimmen), Edge-TTS (kostenlos, Microsoft Neural Voices) und Bark (Suno AI mit Soundeffekten). Die Extension läuft asynchron und blockiert die Textgenerierung nicht. Ideal für Voice-Assistenten und barrierefreie Zugänge.

RAG (Retrieval-Augmented Generation)

Die RAG-Extension bringt Wissensdatenbank-Integration. PDFs, Textdateien oder HTML-Seiten werden automatisch gechunkt, mit SentenceTransformers embeded und in ChromaDB oder FAISS gespeichert. Mit /ask "Frage" durchsucht das LLM deine Dokumente und liefert kontextbezogene Antworten – lokal, datenschutzkonform und ohne Cloud-Anbindung. Ideal für Support-Dokumentationen, technische Handbücher oder vertrauliche Firmendaten.

Whisper (Speech-to-Text)

Basiert auf OpenAIs Whisper large-v3. Per Klick auf das Mikrofon-Icon beginnt die Aufnahme, Whisper transkribiert lokal in 100+ Sprachen (inklusive Deutsch mit hervorragender Erkennungsqualität). Zusammen mit der TTS-Extension entsteht ein vollständiger, sprachgesteuerter KI-Assistent – komplett lokal, keine Cloud-Anbindung.

Extension Funktion Backends
TTS Text-to-Speech, Voice-Cloning XTTS-v2, ElevenLabs, Edge-TTS, Bark
RAG Wissensdatenbank, Q&A auf Dokumente ChromaDB, FAISS, SentenceTransformers
Whisper Speech-to-Text, Transkription Whisper large-v3, turbo

oobabooga auf dem VPS: Produktionsbetrieb

Ein GPU-VPS mit NVIDIA A5000 (24 GB) oder RTX 4090D (24 GB) kostet monatlich 80–200 Euro und ersetzt praktisch jedes Cloud-API-Abo. Für den Produktionsbetrieb empfehlen wir:

  • systemd-Service mit Restart=always für automatische Neustarts
  • Nginx-Reverse-Proxy mit Let's Encrypt SSL vor der API
  • API-Key-Authentifizierung via --api-key oder Nginx Basic Auth
  • Wöchentlicher Cron-Job für git pull und Abhängigkeits-Updates

Häufige Probleme

Problem Lösung
CUDA Out of Memory Niedrigere Quantisierung (Q4 statt Q8), --auto-devices setzen
Modell lädt nicht Falscher Loader-Typ – GGUF braucht llama.cpp, EXL2 braucht ExLlamaV2
Langsame Generierung n-gpu-layers erhöhen, Flash-Attention aktivieren
API gibt 404 Server mit --api starten, Port-Firewall prüfen

Fazit

Die Text Generation WebUI (oobabooga) ist 2026 die führende Open-Source-Plattform für lokale Large Language Models. Mit breiter Format-Unterstützung für GGUF, GPTQ, EXL2, AWQ und HQQ, integriertem LoRA-Management, einer vollständig OpenAI-kompatiblen API und einem vielseitigen Extensions-System deckt sie praktisch jeden Anwendungsfall ab – vom privaten Chat-Assistenten auf dem Desktop bis zur produktiven KI-API auf dem VPS.

Der Einstieg ist einfach: Conda-Umgebung einrichten, Repository klonen, Modell laden, chatten. Fortgeschrittene Nutzer profitieren von LoRA-Adaptern für spezifische Domänen, der API für externe Tools und der RAG-Integration für wissensbasierte Abfragen auf eigenen Dokumenten. Die Kombination aus TTS und Whisper macht oobabooga zudem zu einem vollwertigen, sprachgesteuerten KI-Assistenten – komplett lokal, ohne Cloud-Anbindung.

Ein eigener GPU-VPS mit NVIDIA A5000 oder RTX 4090D ist auf Dauer günstiger als Cloud-LLM-APIs und bietet vollständige Datenkontrolle – ein entscheidender Vorteil für Unternehmen mit Datenschutzauflagen. Wer KI ernsthaft lokal betreiben möchte, kommt 2026 an oobabooga nicht vorbei.
