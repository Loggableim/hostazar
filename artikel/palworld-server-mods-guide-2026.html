Warum Palworld auf einem eigenen Dedicated Server?

Ein eigener Palworld Dedicated Server ist die einzige Lösung, die dir gleichzeitig volle Performance, Admin-Rechte und Mod-Support bietet. Auf offiziellen Serellen oder über das Matchmaking von Pocketpair bist du an Slot-Limits, kurze Session-Zeiten und das vorgegebene Regelwerk gebunden. Auf einem selbst verwalteten Server entscheidest du, wie viele Spieler joinen, wie schnell die Nacht wechselt, wie streng der Tod bestraft wird und welche Mods das Gameplay ergänzen.

Drei Gründe sprechen 2026 besonders für den eigenen Server: Performance – du dimensionierst RAM, CPU und NVMe exakt nach Spielerzahl und Weltgröße. Mods – BepInEx, PalSchema und UEModLoader eröffnen Quality-of-Life-Verbesserungen, Pal-Overhauls und Admin-Werkzeuge, die das Originalspiel nicht liefert. Admin-Rechte – Teleport, Inventar-Manipulation, Spawn-Steuerung und Cheat-Befehle machen Community-Management überhaupt erst möglich.

Systemanforderungen für einen modded Palworld-Server

Mods erhöhen die Anforderungen spürbar – vor allem an RAM und Single-Core-CPU-Leistung. Die folgende Tabelle zeigt erprobte Werte aus 2026er-Teststellungen mit aktivem BepInEx-Setup und Quality-of-Life-Mods:

Szenario Spieler RAM CPU Speicher
Kleine Koop-Runde, wenige QoL-Mods 2–4 8 GB 4 vCPU, hoher Single-Core-Takt 40 GB NVMe
Aktive Freundesgruppe mit Overhaul-Mods 5–12 12–16 GB 6 vCPU, dedizierte Kerne 60 GB NVMe
Große Community + BepInEx + UEModLoader 13–24 20–24 GB 8 vCPU, AMD EPYC / Ryzen 9 80+ GB NVMe
Public Mod-Server, 24/7, viele Basen 25–32 32 GB 8+ dedizierte Kerne, Premium-CPU 120 GB NVMe + Backup-Volume

Wichtig: Single-Core-Leistung schlägt Kern-Anzahl. Palworld skaliert hauptsächlich über den Haupthread der UE5-Engine. Moderne AMD-Epyc-, Ryzen-7- oder Intel-Xeon-VPS-Konfigurationen schlagen Billigangebote mit 16 vCPUs, die auf Atom-Cores basieren.

Dedicated Server auf VPS installieren (Linux & Windows)

Die Installation läuft auf beiden Plattformen identisch ab, lediglich die Systemvorbereitung unterscheidet sich. Grundsätzlich empfehlen wir Linux (Ubuntu 22.04/24.04 LTS), weil es günstiger ist, weniger RAM-Verbrauch hat und sich hervorragend als systemd-Dienst einrichten lässt. Windows lohnt sich nur, wenn du Tools mit GUI einsetzen willst oder bereits eine Windows-Lizenz besitzt.

Linux-Vorbereitung (Ubuntu):

sudo apt update && sudo apt upgrade -y
sudo apt install -y lib32gcc-s1 lib32stdc++6 steamcmd
sudo useradd -m -s /bin/bash palworld
sudo mkdir -p /opt/palworld
sudo chown palworld:palworld /opt/palworld

Windows-Vorbereitung:

  • Visual C++ Redistributable 2022 installieren
  • DirectX End-User Runtime bereitstellen
  • Eigenes Verzeichnis C:\PalworldServer anlegen
  • Windows-Firewall-Regel für UDP 8211 erstellen

SteamCMD-Setup für Palworld

Der offizielle Palworld-Server wird über SteamCMD geladen und aktualisiert. Lege ein eigenes Verzeichnis an und führe folgende Befehle aus:

# Linux
steamcmd +login anonymous \
  +force_install_dir /opt/palworld \
  +app_update 2394010 validate \
  +quit

# Windows (CMD als Admin)
steamcmd.exe +login anonymous ^
  +force_install_dir C:\PalworldServer ^
  +app_update 2394010 validate ^
  +quit

Damit der Server nicht als Root läuft, startest du ihn auf Linux später mit dem angelegten palworld-User. Per systemd-Service sorgst du zusätzlich für automatische Neustarts nach Crashs oder Reboots.

# /etc/systemd/system/palworld.service
[Unit]
Description=Palworld Dedicated Server
After=network.target

[Service]
User=palworld
WorkingDirectory=/opt/palworld
ExecStart=/opt/palworld/PalServer.sh
Restart=on-failure
RestartSec=10
LimitNOFILE=100000

[Install]
WantedBy=multi-user.target

Aktivieren und starten:

sudo systemctl daemon-reload
sudo systemctl enable --now palworld
sudo systemctl status palworld

server.ini Konfiguration erklärt

Die zentrale Konfigurationsdatei für deinen Palworld-Server ist PalWorldSettings.ini (oft nur server.ini genannt). Sie liegt im Verzeichnis /opt/palworld/Pal/Saved/Config/LinuxServer/. Beim ersten Serverstart wird sie automatisch mit Standardwerten erzeugt. Hier ein praxiserprobtes Beispiel für eine gut spielbare Mid-Game-Welt mit aktiver Community:

; /opt/palworld/Pal/Saved/Config/LinuxServer/PalWorldSettings.ini

[/Script/Pal.PalGameWorldSettings]
OptionSettings=(
  ServerName="Hostazar Palworld 2026",
  ServerDescription="Modded Koop-Server, PvE, 16 Slots",
  AdminPassword="AENDERN-MIT-LANGEM-PASSWORT!",
  ServerPassword="",

  ; Welt & Schwierigkeit
  Difficulty=Normal,
  DayTimeSpeedRate=1.0,
  NightTimeSpeedRate=1.0,
  ExpRate=1.5,
  PalCaptureRate=1.5,
  PalSpawnNumRate=1.2,
  PalDamageRateAttack=1.0,
  PalDamageRateDefense=1.0,
  PlayerDamageRateAttack=1.0,
  PlayerDamageRateDefense=1.0,
  PlayerStomachDecreaseRate=1.0,
  PlayerStaminaDecreaseRate=1.0,
  PlayerAutoHPRegeneRate=1.0,
  PlayerAutoHpRegeneRateInSleep=2.0,

  ; Tod & Items
  DeathPenalty=All,                ; None / Item / Equipment / All
  bEnableInvaderEnemy=True,
  DropItemMaxNum=5000,
  DropItemAliveMaxHours=12.0,

  ; Basen & Bauen
  BaseCampMaxNum=128,
  BaseCampWorkerMaxNum=20,
  BuildObjectHpRate=1.0,
  BuildObjectDamageRate=1.0,
  BuildObjectDeteriorationDamageRate=1.0,

  ; Performance
  CollectionObjectHpRate=1.0,
  CollectionObjectRespawnSpeedRate=1.0,
  EnemyDropItemRate=1.0,
  bEnableAimAssistPad=True,
  bEnableAimAssistKeyboard=True,

  ; PvP & Multiplayer
  bEnablePlayerToPlayerDamage=False,
  bEnableFriendlyFire=False,
  bEnableNonLoginPenalty=True,
  NonLoginPenaltyHard=False,

  ; Slots & Netzwerk
  ServerPlayerMaxNum=16,
  bIsUseBackupSaveData=True,
  BackupInterval=30,
  SaveInterval=30
)

Die wichtigsten Parameter im Überblick:

  • DeathPenalty – None (kein Verlust), Item, Equipment, All (komplett). Auf PvE-Servern mit aktiver Community empfehlen wir Item für moderaten Druck.
  • PalSpawnNumRate – Multiplikator für die Spawn-Dichte. Werte zwischen 1.0 und 1.5 sind erfahrungsgemäß sinnvoll. Über 2.0 leidet die Performance.
  • DayTimeSpeedRate / NightTimeSpeedRate – Beschleunigt den Tag-Nacht-Wechsel. Für gemütliche Koop-Runden 1.0, für Speed-Runner 2.0–3.0.
  • ExpRate / PalCaptureRate – Beide gleichmäßig anheben, sonst kippt die Spielbalance. 1.5/1.5 ist ein guter Start.
  • BaseCampMaxNum – Erlaubt mehrere Basen pro Guild. 64–128 ist auf größeren Servern realistisch.
  • ServerPlayerMaxNum – Slot-Anzahl. Performance-kritisch: 16 stabile Slots > 32 ruckelnde.
  • BackupInterval – Automatische Sicherung alle 30 Minuten. In Kombination mit bIsUseBackupSaveData=True ein wirksamer Schutz.

⚠️ Sicherheits-Hinweis: AdminPassword und ServerPassword niemals öffentlich posten. Bei gemieteten Servern prüfen, ob der Anbieter die Werte im Panel einsehbar anzeigt – sonst Spieler-IPs aussperren.

Mods installieren: BepInEx, PalSchema, UEModLoader

Die Modding-Szene für Palworld nutzt BepInEx als universellen Mod-Loader. Damit lassen sich Pal-Stat-Änderungen, UI-Overhauls und Server-seitige Tweaks einspielen, ohne den Spielcode zu patchen. Ergänzend existieren PalSchema für Datenanpassungen und der UEModLoader für umfangreichere Unreal-Engine-Mods.

1. BepInEx installieren (Linux):

cd /opt/palworld
wget https://github.com/BepInEx/BepInEx/releases/latest/download/BepInEx_Unix_x64_5.4.23.3.zip
unzip BepInEx_Unix_x64_5.4.23.3.zip
chmod +x BepInEx/run_bepinex.sh
./BepInEx/run_bepinex.sh PalServer.sh

2. Mods einbinden:

Alle heruntergeladenen .dll-Dateien landen in BepInEx/plugins/. Beim nächsten Start des Servers werden sie automatisch geladen – sichtbar in der Konsole als „Plugin loaded: …“.

3. PalSchema konfigurieren:

PalSchema-Dateien (*.json) liegen unter BepInEx/config/PalSchema/. Jede Datei überschreibt Spawn-Wahrscheinlichkeiten, Pal-Stats, Drops oder Boss-Verhalten. Beispiel:

{
  "PalName": "Lamball",
  "OverrideStats": {
    "HP": 1200,
    "Attack": 95,
    "Defense": 80,
    "CraftSpeed": 100
  },
  "OverrideDropItems": [
    { "ItemName": "Wool", "DropRate": 0.6 }
  ]
}

4. UEModLoader für schwere Mods:

UEModLoader wird zusätzlich zu BepInEx installiert und erlaubt .pak-Mods, die komplette Unreal-Asset-Pakete austauschen – etwa überarbeitete Map-Sektionen, Custom-Pals oder komplette UI-Reskins. Die Dateien landen in Pal/Content/Paks/Mods/.

Crossplay-Konfiguration (Steam + Xbox + Game Pass)

Seit dem Crossplay-Patch 2025 spielen Steam-, Xbox- und Game-Pass-User in denselben Welten. Auf deinem Dedicated Server ist Crossplay standardmäßig aktiv. Wichtig ist, dass die RESTAPI-Plugins korrekt geladen sind und dein Anbieter Multi-Plattform-Listen-Server unterstützt. Bei Nitrado, G-Portal und den meisten deutschen Hostern ist das der Fall.

  • Server-Region EU/US: Spieler aller Plattformen verbinden sich automatisch über den globalen Server-Browser.
  • Invite-Codes: Funktionieren plattformübergreifend – ein Steam-Spieler kann Xbox-Freunde einladen.
  • Voice-Chat: Plattform-intern (Steam Voice / Xbox Party). Eigene Discord-Channel lohnen sich für gemischte Gruppen.
  • Account-Limits: Xbox-Konten benötigen Game-Pass-Abo. Steam-Accounts benötigen nur den Base-Game-Kauf.

🔌 Praxis-Tipp: Bei Problemen mit Crossplay-Login den RCONEnabled=True-Parameter prüfen und sicherstellen, dass der UDP-Port 8211 für alle Plattformen erreichbar ist. Manche Internet-Provider (vor allem Mobilfunk) blockieren UDP-Ausgang – da hilft ein Wechsel auf eine Festnetz-Leitung oder VPN mit UDP-Support.

Backup-Strategie für Savegames

Palworld-Welten können 1–4 GB groß werden – und sie enthalten hunderte Stunden Spielzeit. Eine mehrstufige Backup-Strategie schützt vor Datenverlust:

  1. Automatisches In-Game-Backup: bIsUseBackupSaveData=True + BackupInterval=30 in der server.ini erzeugt alle 30 Minuten Snapshots im Savegame-Ordner.
  2. Stündliches Cron-Backup: Auf Linux per cronjob, der den Saved/SaveGames/-Ordner in ein Backup-Verzeichnis kopiert und ältere Stände nach 7 Tagen löscht.
  3. Tägliche Off-Site-Sicherung: Per rclone auf einen S3-kompatiblen Speicher (Hetzner Storage Box, Backblaze B2, Wasabi) oder auf einen zweiten VPS.
  4. Vor jedem Update: Manueller Snapshot. Bei großen Patches hat Pocketpair schon mehrfach Welten zerschossen – ohne Backup ist die Welt dann weg.
  5. Restore-Test: Mindestens einmal pro Quartal ein Backup auf einem Testserver zurückspielen und prüfen, ob alle Basen, Pals und Inventare vollständig sind.
# /usr/local/bin/palworld-backup.sh
#!/bin/bash
SRC=/opt/palworld/Pal/Saved/SaveGames
DST=/backups/palworld/$(date +%Y%m%d-%H%M)
mkdir -p "$DST"
tar czf "$DST/save.tar.gz" -C "$SRC" .
find /backups/palworld -type d -mtime +7 -exec rm -rf {} +

Performance-Tuning: Tickrate, Memory, Garbage Collection

Mit diesen Stellschrauben holst du 10–25 % mehr Performance aus deinem Server – wichtig gerade auf modded Welten mit vielen Pals und Basen:

  • Java-/UE-Startparameter: Auf Linux dem PalServer.sh -useallmats -NoVerifyGC -NoAsyncLoadingThread hinzufügen (nur für fortgeschrittene Admins).
  • Tickrate beobachten: Live-Status mit /info. Werte unter 30 ms/Frame sind ideal, alles über 60 ms deutet auf Engpässe hin.
  • Memory-Limit setzen: Auf Linux-VPS per systemd-Unit MemoryMax=22G definieren, damit der OOM-Killer nicht den ganzen Server abschießt.
  • Swapiness reduzieren: sysctl vm.swappiness=10 verhindert aggressives Auslagern auf SSD.
  • Disk-I/O: NVMe-VPS bevorzugen, SATA-SSD ist in 2026er-Setups deutlich langsamer.
  • Mod-Last reduzieren: Mods wie „Palworld Plus" (Performance-Pack) aktivieren – sie optimieren KI- und Streaming-Logik.

🛠️ Monitoring-Pflicht: Auf Uptime Kuma oder Prometheus + Grafana setzen, um CPU, RAM und Tickrate dauerhaft im Blick zu behalten. Wir haben das in unserem Prometheus-Grafana-Guide und im Uptime-Kuma-Tutorial detailliert beschrieben.

Häufige Probleme: Verbindungsabbrüche, Mod-Inkompatibilität, Anti-Cheat

Drei Fehlerklassen treten auf modded Palworld-Servern regelmäßig auf – mit den passenden Workouts löst du sie in unter zehn Minuten:

1. Verbindungsabbrüche & Timeout-Fehler
Häufige Ursache: UDP-Port 8211 durch Firewall oder NAT geblockt. Auf VPS mit ufw oder firewalld freischalten. Bei Home-Hosting Port-Forwarding am Router prüfen. Anti-Cheat kickt gelegentlich Spieler mit aggressiven VPNs – in dem Fall IPv4-only aktivieren.

2. Mod-Inkompatibilität nach Update
Pocketpair patcht alle 4–6 Wochen. Mods hinken oft 1–14 Tage hinterher. Workaround: Server auf der letzten stabilen Version pinnen, bis alle Mods aktualisiert wurden. Mehrere Mod-Loader (BepInEx + UEModLoader) können sich gegenseitig blocken – die Reihenfolge in chainload beachten.

3. Anti-Cheat-Bans trotz privater Welt
Auch auf privaten Dedis läuft der Anti-Cheat. Einige Mods – besonders Speed-Hacks, Godmode oder Item-Duplikation – triggern ihn. Server-Admin-Tools filtern das sauber heraus. Wer dauerhaft cheaten will, sollte einen reinen Offline- bzw. LAN-Server betreiben.
