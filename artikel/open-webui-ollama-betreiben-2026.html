Open WebUI mit Ollama betreiben – Der bessere ChatGPT-Klon 2026

Open WebUI ist die schlankere, schnellere und vor allem datenschutzkonforme Alternative zu ChatGPT. In Kombination mit Ollama betreibst du deinen eigenen KI-Assistenten auf deinem Server oder VPS – mit vollem Zugriff auf lokale LLMs wie Llama 3, DeepSeek, Mistral & Co. Wir zeigen dir die komplette Installation, Konfiguration für mehrere Benutzer und wie du RAG-Pipelines für eigene Dokumente aufsetzt.

Einleitung: Warum Open WebUI + Ollama die bessere Alternative zu ChatGPT ist

ChatGPT hat die Welt im Sturm erobert – aber der Hype bringt auch Schattenseiten mit sich: Deine Daten wandern auf US-Server, die monatliche Abo-Gebühr für ChatGPT Plus liegt bei 20 US-Dollar (knapp 22 € im Monat), und bei hohem Traffic bremst OpenAI die Antwortgeschwindigkeit bewusst aus. Hinzu kommt, dass GPT-4o und o1 in der EU teils nur eingeschränkt verfügbar sind oder bestimmte Features (Advanced Voice, Memory) regionsabhängig deaktiviert wurden. Wer seine sensiblen Daten nicht in fremde Hände geben oder auf die volle Kontrolle über das verwendete Modell nicht verzichten möchte, braucht eine Alternative – und die heißt Open WebUI in Kombination mit Ollama.

Open WebUI ist eine vollständig selbst gehostete Chat-Oberfläche, die bewusst an ChatGPTs Look-and-Feel angelehnt ist – mit Namenskonvention wie "Neuer Chat", Dunkelmodus, Markdown-Support, Code-Highlighting und einem sauber getrennten Chat-Verlauf pro Benutzer. Im Hintergrund läuft Ollama als LLM-Runtime, die lokale Sprachmodelle wie Llama 3, Mistral, Qwen 2.5, DeepSeek oder Phi-4 verwaltet. Das Besondere: Beide Komponenten laufen auf deinem eigenen VPS, deinem Homeserver oder sogar deinem Gaming-PC – ohne Internetverbindung, ohne Datenabfluss und ohne monatliche Lizenzkosten.

Dieser Guide führt dich Schritt für Schritt durch die Installation und optimale Konfiguration beider Systeme – vom Setup der Docker-Umgebung über die Ollama-Integration bis hin zum Multi-User-Betrieb und dem Aufsetzen von RAG-Pipelines (Retrieval-Augmented Generation) für deine eigenen Dokumente.

Was ist Open WebUI?

Open WebUI (früher bekannt als Ollama WebUI) ist ein Open-Source-Chat-Interface, das speziell für den Betrieb mit lokalen Large Language Models entwickelt wurde. Es ist in TypeScript mit Svelte (Frontend) und Python mit FastAPI (Backend) geschrieben und bietet gegenüber der puren Ollama-Kommandozeile eine ganze Reihe von Vorteilen:

  • ChatGPT-ähnliche Oberfläche: Thread-basierte Chats, Markdown-Rendering, Syntax-Highlighting für über 50 Sprachen, Datei-Uploads per Drag & Drop
  • Multi-User-Support: Registrierung, Login, Benutzerverwaltung mit Admin-Rollen, Workspace-Isolation pro Team oder Abteilung
  • RAG-Pipelines: Eigene PDFs, Word-Dokumente oder Textdateien hochladen und per Vektor-Suche in Chats einbinden
  • Modell-Wechsel: Mehrere Modelle parallel verwalten und im Chat per Dropdown umschalten – sogar mit parallelen Chats in verschiedenen Modellen gleichzeitig
  • Plugin-System: Erweiterbar über Tools und Actions (Web Search, Code Execution, Image Generation)
  • Mobile PWA: Als Progressive Web App auf dem Smartphone installierbar – inklusive Push-Benachrichtigungen

Das Projekt wird aktiv auf GitHub entwickelt (github.com/open-webui/open-webui) und hat inzwischen mehr als 60.000 Stars. Die Community ist groß, die Release-Frequenz hoch – Updates erscheinen im Schnitt alle zwei bis vier Wochen. Lizenziert unter MIT, also frei für kommerzielle und private Nutzung ohne Einschränkungen.

Voraussetzungen – Was du brauchst

Bevor es losgeht, ein Überblick über die benötigten Komponenten. Die Wahl des Hosts richtet sich maßgeblich nach den Modellen, die du ausführen möchtest:

  • Server oder VPS – mindestens 8 GB RAM für kleinere Modelle (Llama 3 8B, Mistral 7B), ab 32 GB RAM für Llama 3 70B / DeepSeek V3. Empfehlung: dedizierter AWS-VPS oder Hetzner Cloud (CX32/CX52). GPU-Unterstützung (NVIDIA CUDA) empfohlen, aber nicht zwingend erforderlich – CPU-Inferenz funktioniert, ist aber deutlich langsamer.
  • Docker & Docker Compose – Open WebUI und Ollama laufen als Container. Docker Compose (V2) ist der empfohlene Weg für das Setup.
  • ca. 20 GB freier Speicher pro Modell – Llama 3 8B benötigt rund 16 GB, DeepSeek R1:Disti 32B rund 20 GB.
  • Optional: NVIDIA GPU – mit CUDA 12.4+ und nvidia-container-toolkit. Beschleunigt die Token-Generierung um den Faktor 3–10.

Betriebssystem: Ubuntu 22.04/24.04 LTS wird empfohlen (höchste Kompatibilität mit NVIDIA-Treibern und Docker), aber auch Debian 12, Arch Linux, macOS (Apple Silicon) und Windows via WSL2 sind unterstützt.

Schritt 1: Docker & Docker Compose installieren

Falls Docker noch nicht auf deinem Server läuft, installierst du es mit einer Handvoll Befehlen. Auf Ubuntu 24.04 LTS sieht der Standard-Weg so aus:

# System aktualisieren
sudo apt update && sudo apt upgrade -y

# Docker-Paketquellen hinzufügen
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Docker-Dienst starten
sudo systemctl enable --now docker

# Docker Compose (Plugin) prüfen
docker compose version

Das nvidia-container-toolkit (für GPU-Nutzung) installierst du im Anschluss:

# NVIDIA Container Toolkit
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt update && sudo apt install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker

Schritt 2: Ollama mit Docker Compose starten

Der einfachste Weg zu einem sauberen Setup führt über eine gemeinsame Docker-Compose-Datei. Erstelle ein neues Verzeichnis und darin die Datei docker-compose.yml:

version: '3.8'
services:
  ollama:
    image: ollama/ollama:latest
    container_name: ollama
    restart: unless-stopped
    ports:
      - "11434:11434"
    volumes:
      - ./ollama_data:/root/.ollama
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: open-webui
    restart: unless-stopped
    ports:
      - "3000:8080"
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
      - WEBUI_AUTH=true
      - WEBUI_NAME=Mein KI-Assistent
      - DEFAULT_MODELS=llama3:latest
      - ENABLE_SIGNUP=false
    volumes:
      - ./open-webui_data:/app/backend/data
    depends_on:
      - ollama
    extra_hosts:
      - "host.docker.internal:host-gateway"

Dann startest du die Container:

docker compose up -d

Die Open WebUI ist jetzt unter http://DEINE_SERVER_IP:3000 erreichbar. Der Admin-Account wird beim ersten Aufruf angelegt – nachdem du ENABLE_SIGNUP=false gesetzt hast, ist die Registrierung geschlossen und nur du als Admin kannst neue Benutzer anlegen.

Schritt 3: Modelle mit Ollama herunterladen

Jetzt kommt der spannende Teil: Du holst dir die passenden Modelle auf deinen Server. Ollama verwaltet Modelle über eine einfache Kommandozeile – wahlweise direkt im Container oder via API:

# Im Container
docker exec -it ollama ollama pull llama3:latest
docker exec -it ollama ollama pull deepseek-r1:7b
docker exec -it ollama ollama pull mistral:latest
docker exec -it ollama ollama pull qwen2.5:7b

# Verfügbare Modelle anzeigen
docker exec -it ollama ollama list

Alternativ kannst du Modelle auch direkt über die Open-WebUI-Oberfläche laden: Gehe zu Einstellungen → Modelle → "Modell aus Ollama-Bibliothek hinzufügen". Die Oberfläche listet alle verfügbaren Modelle aus dem Ollama-Katalog auf und lädt sie auf Knopfdruck herunter.

Welches Modell für dich das richtige ist, hängt von deiner Hardware und deinem Einsatzzweck ab. Eine kleine Orientierungshilfe:

Schritt 4: Open WebUI einrichten & konfigurieren

Nach dem ersten Login als Admin erwarten dich einige wichtige Konfigurationen, die den Betrieb erst richtig rund machen:

Benutzerverwaltung aktivieren: Gehe zu Admin-Panel → Benutzer. Hier siehst du alle registrierten Accounts. Du kannst neue Benutzer manuell anlegen, Passwörter zurücksetzen oder Zugriffsrechte entziehen. Für Teams empfiehlt sich die Integration mit OIDC (Keycloak, Authelia, Authentik) oder LDAP – beides wird nativ von Open WebUI unterstützt.

RAG konfigurieren: Unter Admin-Panel → RAG stellst du die Chunk-Größe (Standard: 1000 Zeichen) und den Overlap (200 Zeichen) ein. Als Embedding-Modell kannst du entweder ein lokales Ollama-Modell nutzen (z. B. nomic-embed-text) oder einen externen API-Provider. Die Vektordatenbank wird automatisch von Open WebUI mit ChromaDB verwaltet – keine manuelle Einrichtung nötig.

Modell-Presets: Definiere Standard-Parameter pro Modell – Temperatur (0.7 für kreativ, 0.1 für faktenbasierte Antworten), Max-Tokens (2048 als Standard), System-Prompt (z. B. "Du bist ein hilfreicher deutscher Assistent") und Context-Length (8192, 16384 oder 32768 Tokens je nach Modell).

Web-Suche: Open WebUI kann bei Bedarf das Web durchsuchen, um aktuelle Informationen in die Antwort einzubinden. Aktiviere unter Admin-Panel → Tools die Web-Suche und konfiguriere einen Search-API-Key (Google Custom Search, SearXNG oder Bing Search API).

Schritt 5: Multi-User-Modus für Teams und Familien

Einer der größten Vorteile von Open WebUI gegenüber der simplen Ollama-CLI ist der integrierte Multi-User-Modus. Du kannst beliebig viele Benutzer anlegen, die jeweils eigene Chat-Verläufe, Workspaces und Modellberechtigungen haben:

  • Admin-Rolle: Volle Kontrolle – Benutzer verwalten, Systemeinstellungen, alle Chats einsehen, Modelle hinzufügen/löschen
  • User-Rolle: Eigene Chats, hochgeladene Dokumente, Modellauswahl – kein Zugriff auf Admin-Bereich oder fremde Chats
  • Read-Only-Rolle: Nur Chat-Verlauf einsehbar, keine neuen Chats startbar – nützlich für externe Gäste oder Viewer

Für Unternehmen mit vielen Benutzern empfiehlt sich die Anbindung an einen OIDC-Provider (z. B. Keycloak oder Authelia). Die Konfiguration erfolgt über Environment-Variablen in der docker-compose.yml:

environment:
  - WEBUI_AUTH=true
  - ENABLE_SIGNUP=false
  - OAUTH_PROVIDER=google  # oder: microsoft, github, oidc
  - OAUTH_CLIENT_ID=xxxxxxxx
  - OAUTH_CLIENT_SECRET=yyyyyyyy

Nach dem Neustart können sich Benutzer per Single-Sign-On (SSO) anmelden – und du hast die volle Kontrolle darüber, wer auf deine KI-Instanz zugreifen darf.

RAG-Pipelines: Eigene Dokumente in die KI einbinden

Die Killer-Funktion von Open WebUI ist die eingebaute RAG-Pipeline. Anders als ChatGPT, das nur auf seinem Trainingswissen basiert (Stand: Mai 2026), kannst du bei Open WebUI eigene Dokumente hochladen und im Chat als Wissensbasis nutzen. Der Ablauf im Hintergrund:

  1. Upload: Du lädst PDF, DOCX, TXT (oder sogar ganze Ordner) über die Oberfläche hoch. Die maximale Dateigröße lässt sich im Admin-Panel konfigurieren (Standard: 100 MB).
  2. Chunking & Embedding: Die Datei wird in semantische Abschnitte zerlegt (Chunk-Größe einstellbar) und jedes Chunk wird mit einem Embedding-Modell (z. B. nomic-embed-text) in numerische Vektoren umgewandelt.
  3. Vektor-Speicher: Die Embeddings landen in einer lokalen ChromaDB-Datenbank. Pro Workspace kann es einen eigenen Vektorindex geben, sodass Teams nur auf ihre eigenen Dokumente zugreifen.
  4. Retrieval: Bei jeder Chat-Nachricht wird automatisch der Vektorindex durchsucht, die 3–5 relevantesten Chunks ausgewählt und als Kontext an das Modell übergeben.

Das Ergebnis: Dein lokales LLM kann plötzlich Fragen zu deinen internen Dokumenten beantworten – ohne dass du die Dokumente jemals an einen externen Server senden musst. Perfekt für Handbücher, Verträge, wissenschaftliche Paper oder interne Wikis.

Hardware-Empfehlung: Der richtige Server für dein KI-Setup

Je nachdem, welche Modelle du betreiben willst, unterscheidet sich die benötigte Hardware massiv. Hier eine grobe Einordnung, welcher Server für welches Szenario die richtige Wahl ist:

  • Einsteiger (Llama 3 8B, Mistral 7B): Ein VPS mit 8 GB RAM und 4 vCPUs reicht aus – z. B. Hetzner CX22 (ca. 5,89 €/Monat) oder Netcup VPS 1000 (ca. 12 €/Monat). CPU-Inferenz liefert hier 5–15 Tokens pro Sekunde.
  • Fortgeschritten (DeepSeek 32B, Qwen 32B): 16–24 GB RAM, 8 vCPUs – Hetzner CX52 (ca. 18 €/Monat) oder ein günstiger Dedicated Server mit Ryzen 5. Mit GPU (RTX 3060 12 GB) erreichst du 30–60 Tokens/s.
  • Pro (Llama 3 70B, Mixtral 8x22B): Dedizierter Server mit 64+ GB RAM und NVIDIA RTX 4090 (24 GB) oder besser A6000 (48 GB). Server-Miete ab ca. 150 €/Monat bei Hetzner oder Contabo.

Ein wichtiger Tipp für den Einstieg: Starte mit einem kleinen Modell wie Llama 3 8B oder Qwen 2.5 7B auf einem günstigen VPS (8–16 GB RAM). Wenn du merkst, dass die Antwortqualität für deine Anwendungsfälle reicht, hast du kaum Investitionskosten gehabt. Für erste Tests reicht sogar ein Raspberry Pi 5 mit 8 GB RAM – darauf läuft Phi-4 oder Qwen 2.5 7B in 4-Bit-Quantisierung (über Ollama: qwen2.5:7b-q4_K_M) flüssig genug für den gelegentlichen Gebrauch.

Sicherheitshinweise für den Produktivbetrieb

Wenn Open WebUI über das Internet erreichbar sein soll (z. B. für dein Team), gibt es ein paar wichtige Sicherheitsvorkehrungen zu beachten:

  • Reverse Proxy mit SSL: Setze Nginx oder Caddy als Reverse Proxy vor Open WebUI, der automatisch Let's Encrypt-Zertifikate ausstellt. Beispiel-Nginx-Config gibt es im offiziellen Wiki.
  • Firewall: Port 3000 und Port 11434 nur aus dem internen Netz (Docker-Netzwerk oder Tailscale/Wireguard) erreichbar machen. Niemals Port 11434 direkt ins Internet freigeben!
  • Authentifizierung erzwingen: Setze WEBUI_AUTH=true und ENABLE_SIGNUP=false. Nur manuell angelegte Benutzer erhalten Zugriff.
  • Regelmäßige Updates: Open WebUI und Ollama erscheinen im Wochenrhythmus. Führe regelmäßig docker compose pull && docker compose up -d aus, um Sicherheitslücken zu schließen.
  • Raten-Limiting: Mit einem vorgeschalteten Nginx lässt sich die Anzahl der Requests pro Minute pro IP begrenzen – wichtig, um API-Missbrauch zu verhindern.

Fazit

Open WebUI und Ollama bilden das derzeit beste Open-Source-Duo, um einen eigenen KI-Assistenten auf dem Server zu betreiben. Die Installation via Docker Compose ist in 15 Minuten erledigt, die laufenden Kosten beschränken sich auf die Server-Miete – kein monatliches Abo, keine Datenflucht ins Ausland und volle Kontrolle über die verwendeten Modelle. Der Multi-User-Modus macht die Lösung teamtauglich, die integrierten RAG-Pipelines verwandeln deinen Server in eine lokale Wissensdatenbank mit KI-Suche.

Für den Einstieg reicht ein VPS mit 8 GB RAM und 4 vCPUs – etwa die Hetzner Cloud CX22 (ab 5,89 €/Monat) oder ein Netcup VPS 1000 (ab 11,99 €/Monat). Wenn du größere Modelle wie Llama 3 70B betreiben möchtest, empfehlen wir einen Dedicated Server mit NVIDIA-GPU – die Investition amortisiert sich nach wenigen Monaten gegenüber einem ChatGPT-Team-Abo schnell.

Wer einen Schritt weitergehen möchte: Open WebUI unterstützt mittlerweile auch Image Generation via ComfyUI, Speech-to-Text (Whisper) und Text-to-Speech (TTS). Dein selbst gehosteter KI-Assistent kann dann nicht nur Texte schreiben, sondern auch Bilder generieren, Dokumente transkribieren und per Sprache antworten – komplett lokal und datenschutzkonform. Das ist mehr, als ChatGPT Plus aktuell bieten kann.
